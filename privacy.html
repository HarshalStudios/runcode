1. Introduction

RunCode ("we", "our", or "us") is an Android application designed for students, developers, and programming enthusiasts to write, edit, organize, and execute source code. This Privacy Policy explains what information we collect, how it is used, and how we protect your privacy when you use the RunCode application.

2. Your Projects and Files

Your source code, projects, folders, and programming files remain your property. RunCode stores your workspace locally on your Android device using Android's Storage Access Framework (SAF).

The application can only access files and folders that you explicitly choose through Android's system file picker.

  • Local Storage — Your projects remain stored on your own device.
  • User Controlled Access — RunCode only accesses files or folders you explicitly select.
  • No Background File Scanning — We never scan unrelated files stored on your device.
  • Your Ownership — You retain full ownership of every project and source file you create.

3. Cloud Code Execution

When you press the Run button, RunCode securely sends your source code, selected programming language, and any standard input you provide to a remote code execution service for compilation and execution.

This transmission occurs only when you explicitly choose to execute your code. The request is sent securely using HTTPS encryption.

RunCode does not associate execution requests with your identity, account, or personal information. We do not permanently store your source code on our own servers.

4. Information We Collect

RunCode collects only the minimum information necessary to improve application quality, reliability, and performance.

  • Anonymous application usage analytics
  • Crash reports
  • Performance diagnostics
  • Application version
  • Android version
  • Device model
  • Advertising identifiers used by Google services

We do not collect your personal documents, contacts, messages, photos, passwords, or locally stored programming projects.

5. Third-Party Services

RunCode relies on trusted third-party services to provide core functionality.

  • Judge0 Code Execution Service
    Used to compile and execute source code submitted by the user. When code execution is requested, your source code, selected programming language, and standard input are securely transmitted to the execution service solely to generate execution results.
  • Firebase Analytics
    Used to understand anonymous application usage and improve future versions of RunCode.
  • Firebase Crashlytics
    Used to receive anonymous crash reports and diagnostics to improve application stability.
  • Google AdMob
    Displays advertisements in the free version of RunCode. Google may process advertising identifiers according to Google's own Privacy Policy.
  • Google Play Billing
    Processes optional in-app purchases securely through Google Play. RunCode never receives or stores your payment information.

6. Permissions

RunCode requests only the permissions necessary for its intended functionality.

  • Internet
    Required for cloud code execution, analytics, crash reporting, advertisements, and optional online services.
  • Storage Access Framework (SAF)
    Used only after you explicitly select files or folders to open or save.

RunCode does not request unnecessary or invasive Android permissions.

7. Advertising

The free version of RunCode may display advertisements provided by Google AdMob. Google may collect limited device identifiers and advertising information according to Google's own Privacy Policy.

RunCode does not sell your personal information to advertisers.

8. Data Security

Protecting your projects and privacy is one of our highest priorities.

  • Your workspace files remain stored locally on your device.
  • Network communication uses secure HTTPS encryption.
  • RunCode follows Android security best practices.
  • No user account is required to use RunCode.
  • We do not maintain a cloud database of your programming projects.

9. Children's Privacy

RunCode is intended for students, educators, developers, and programming learners. The application does not knowingly collect personal information from children.

We may update this Privacy Policy from time to time. Any future changes will be published on this page together with the updated revision date.
